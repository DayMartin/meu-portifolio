Pipeline Jenkins com Verificação Contínua SonarQube

Jenkins CI/CD • SonarQube Scanner • Quality Gates • Docker • Git Integrations

Ambiente Corporativo (Tizza)

Problema

Deploys manuais geram quebras frequentes em produção devido a branches desatualizadas, bugs ocultos introduzidos nas features e ausência total de verificação estática de segurança e cobertura mínima de testes unitários.

Solução e Arquitetura

Configuração de uma esteira automatizada no Jenkins que intercepta pull requests nas branches principais do repositório Git, executa testes de integração dentro de contêineres Docker efêmeros e submete o código à varredura estática profunda de vulnerabilidades e bugs no SonarQube Server.

Decisões de Engenharia

  • Estruturação de Quality Gates Rígidos: Se o código analisado apresentar débitos técnicos severos (code smells), vulnerabilidades expostas (Vulnerability Hotspots) ou cobertura de testes inferior a 80%, a esteira interrompe o pipeline e impede o deploy (Quality Gate Failure).
  • Provisionamento de Build Agents Isolados: Para evitar concorrência e lentidão nas builds, configurei o Jenkins para disparar containers Docker dinâmicos como agentes de execução, que são totalmente limpos e destruídos após o término do pipeline.

Resultado

Automatização e blindagem do fluxo de entrega contínua corporativo. Redução expressiva de incidentes em ambiente produtivo, maior consistência nos padrões de código e rastreabilidade total de bugs antes da fase de homologação.

Alta Disponibilidade

Arquitetura AWS com Auto Scaling & Balanceamento de Carga

AWS EC2 • Elastic Load Balancer (ALB) • Auto Scaling Groups • AWS CloudWatch Alerts

Problema

Instabilidades recorrentes causadas por picos imprevisíveis de tráfego de usuários. Uma máquina EC2 única sofria gargalos de CPU e memória, resultando em lentidão geral ou indisponibilidade completa do serviço.

Solução e Arquitetura

Desenho e provisionamento de uma arquitetura baseada em resiliência horizontal. O tráfego de entrada é distribuído de forma equilibrada por um Application Load Balancer (ALB) que monitora a saúde das instâncias subjacentes e direciona requisições apenas para instâncias saudáveis. Um grupo de Auto Scaling dimensiona dinamicamente a quantidade de EC2 ativas com base nas métricas de telemetria coletadas.

Decisões de Engenharia

  • Políticas de Escalonamento Dinâmico por Telemetria: Configurei alarmes do AWS CloudWatch integrados ao Auto Scaling. O sistema inicia o provisionamento de novos servidores quando o uso médio de CPU ultrapassa 70% por mais de 3 minutos, e encerra servidores adicionais quando a carga média recua abaixo de 30% (otimização financeira de recursos).
  • Launch Templates Versionados: Padronização do ambiente via templates de inicialização contendo imagens customizadas pré-instaladas, reduzindo o tempo de boot de novas instâncias para menos de 60 segundos.

Resultado

Infraestrutura elástica tolerante a falhas capaz de suportar picos bruscos de acessos sem interrupção de serviço e com provisionamento e liberação automatizada de recursos AWS em tempo real.

Segurança de Rede

Arquitetura de Rede Privada Isolada (VPC AWS)

AWS VPC • Public/Private Subnets • NAT Gateway • Route Tables • Security Groups • NACLs

Problema

Exposição pública de bancos de dados relacionais e microsserviços de backend diretamente na internet, aumentando drasticamente a superfície de ataque e o risco de vazamento de dados sensíveis ou invasões.

Solução e Arquitetura

Projeto e execução de uma infraestrutura de rede robusta e isolada (Virtual Private Cloud). O ambiente isola a camada pública da privada.

Decisões de Rede & Acesso

  • Sub-redes Públicas vs Privadas: Apenas o Load Balancer e gateways de entrada residem em sub-redes públicas conectadas ao Internet Gateway. A API de backend e o banco de dados RDS residem em sub-redes totalmente privadas, sem endereços de IP públicos disponíveis.
  • Roteamento NAT Gateway Seguro: Servidores em redes privadas que demandam atualizações externas acessam a internet estritamente de forma unidirecional através de um NAT Gateway provisionado na zona pública.
  • Firewalls em Camadas (Security Groups & NACLs): Bloqueio granular de tráfego. O banco de dados MySQL no RDS aceita conexões estritamente originadas pelo grupo de segurança associado às EC2 da API, blindando a camada de persistência.

Resultado

Topologia de rede em nuvem desenhada sob princípios de menor privilégio e segurança rigorosa, protegendo completamente os dados sensíveis corporativos contra invasões diretas ou varreduras de portas externas.

Serverless Architecture

API Serverless para Gestão de Arquivos no S3

AWS Lambda • Amazon API Gateway • Amazon S3 • AWS IAM Permissions • Python

Problema

Manter servidores ligados 24 horas por dia apenas para lidar com picos sazonais de upload, leitura e exclusão de arquivos gera custos computacionais elevados (ociosidade de EC2) e complexidade de manutenção de patches de servidores.

Solução e Arquitetura

Desenvolvimento de uma API REST baseada inteiramente em arquitetura serverless. O fluxo é exposto por endpoints configurados no Amazon API Gateway que invocam funções executadas sob demanda no AWS Lambda (em Python) para realizar operações de persistência e gerenciamento no bucket do Amazon S3.

Decisões de Engenharia

  • Políticas Granulares de Permissão IAM: A função Lambda assume uma Execution Role que limita suas ações especificamente às permissões `s3:GetObject`, `s3:PutObject` e `s3:DeleteObject` dentro do bucket dedicado, obedecendo ao princípio de menor privilégio.
  • Presigned URLs para Alta Performance: Para downloads eficientes de grandes arquivos, a Lambda gera links temporários assinados eletronicamente e os devolve para o cliente, permitindo que a aplicação faça o download diretamente do storage S3 sem passar tráfego de dados pesado pela API Gateway.

Resultado

Estrutura computacional de custo zero para recursos ociosos, escalando infinitamente e de forma totalmente gerenciada pela AWS, suportando uploads e consultas simultâneas sem latência ou perdas de arquivos.
